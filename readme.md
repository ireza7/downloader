# 📥 دانلودر فایل با ربات بله و گیتهاب

<div dir="rtl">

**فایل از هر لینکی دانلود کن، مستقیم توی ریپازیتوری گیتهاب ذخیره کن.**  
دو راه برای دانلود داری:  
۱. ویرایش فایل `trigger_download.txt`  
۲. ربات بله (ربات تلگرام با اندپوینت بله)

> این پروژه در اصل برای تلگرام نوشته شده بود؛ فقط **اندپوینت** رو به بله تغییر دادیم.  
> نام متغیر همون `TELEGRAM_TOKEN` مونده، ولی توکن بله رو توش می‌ذارید.

---

## ✨ امکانات جدید

- **نام واقعی فایل**: قبل از دانلود، اسم فایل از هدر `Content-Disposition` یا URL نهایی (بعد از Redirect) گرفته می‌شود.
- **دانلود تکه‌ای (Chunked)**: اگر سرور پشتیبانی کند، فایل‌های بزرگ به ۴ بخش موازی تقسیم و با سرعت بالاتر دانلود می‌شوند.
- **کش ماژول‌های Go**: اجرای ورک‌فلو کمتر از ۱ ثانیه طول می‌کشد.
- **دستور `/cancel`**: اگر اشتباهی لینک فرستادید، در همان چرخه ۵ دقیقه‌ای می‌توانید دانلود را لغو کنید.
- **دانلود موازی فایل‌ها**: تا ۵ فایل هم‌زمان.
- **سه حالت ذخیره**: ساده، همه در یک zip، هر فایل در zip جداگانه.

---

## 🚀 راه‌اندازی در ۴ مرحله

### ۱. پروژه را Fork کنید
دکمه‌ی **Fork** را بزنید تا یک نسخه در اکانت خودتان داشته باشید.

### ۲. توکن بله را تنظیم کنید
در بله یک ربات بسازید و توکن آن را کپی کنید.  
به مسیر **Settings → Secrets and variables → Actions** بروید و یک **Secret** جدید با مشخصات زیر ایجاد کنید:

| نام              | مقدار |
|------------------|-------|
| `TELEGRAM_TOKEN` | توکن ربات بله |

### ۳. دسترسی‌های ورک‌فلو را فعال کنید
در همان قسمت **Settings → Actions → General**، گزینه‌ی **Workflow permissions** را روی **Read and write permissions** قرار دهید.

### ۴. ورک‌فلو را اجرا کنید
به تب **Actions** بروید و در صورت نیاز تأیید کنید.  
روی ورک‌فلو `Telegram Bot (5min Poll)` کلیک کنید و **Run workflow** را بزنید.  
از این به بعد ربات هر ۵ دقیقه پیام‌ها را چک می‌کند.

## 💬 استفاده از ربات بله

| دستور / پیام               | کار |
|----------------------------|-----|
| یک لینک                    | همان فایل ساده ذخیره می‌شود |
| چند لینک                   | همه در یک فایل zip قرار می‌گیرند |
| `/simple` + لینک‌ها        | دانلود همه‌ی لینک‌ها به‌صورت ساده |
| `/zipall` + لینک‌ها        | همه‌ی فایل‌ها در یک zip |
| `/zipeach` + لینک‌ها       | هر فایل در یک zip جداگانه |
| `/list`                    | لیست فایل‌های دانلود شده در ریپو |
| `/cancel`                  | لغو آخرین درخواست دانلود (فقط در همان ۵ دقیقه) |
| `/help`                    | راهنما |

پس از ارسال لینک، ربات پیام «در حال دانلود...» می‌فرستد و سپس لینک مستقیم فایل را از `raw.githubusercontent.com` برمی‌گرداند.

## 📝 استفاده از فایل `trigger_download.txt`

فایل `trigger_download.txt` در ریشه‌ی پروژه سه بخش دارد:

```
# اگر لینک های خود را در زیر این کامنت وارد کنید فایل ها بدون تغییر به صورت ساده ذخیره میشوند

# اگر لینک هارا زیر این کامنت وارد کنید همه فایل ها در یک فایل فشرده زیپ ذخیره میشوند

# اگر لینک هارا زیر این کامنت وارد کنید هر فایل به صورت یک فایل فشرده زیپ ذخیره میشوند
```

لینک‌های خود را **زیر کامنت مربوطه** بنویسید (هر خط یک لینک) و سپس **Commit changes** را بزنید.  
فایل‌ها به‌طور خودکار دانلود و در پوشه‌ی `downloads/` ذخیره می‌شوند. فایل محرک به حالت اولیه باز می‌گردد.

> ❗ فقط لینک‌هایی که زیر یک بخش قرار بدهید با آن حالت ذخیره می‌شوند. می‌توانید هم‌زمان از چند بخش استفاده کنید.

## ⚙️ توضیحات فنی

- **دانلود تکه‌ای موازی**: اگر سرور فایل از `Range` پشتیبانی کند، فایل به ۴ تکه تقسیم و هم‌زمان دریافت می‌شود – در غیر این صورت دانلود عادی انجام می‌شود.
- **نام فایل واقعی**: ابتدا هدر `Content-Disposition` (مثلاً `attachment; filename="real.pdf"`) بررسی می‌شود، سپس نام از URL نهایی (پس از Redirectها) استخراج می‌شود.
- **اجرای سریع**: به لطف کش ماژول‌های Go، هر بار که ربات چک می‌شود، تنها چند ثانیه طول می‌کشد.
- **مصرف ماهانه**: با چرخه‌ی ۵ دقیقه‌ای، تقریباً ۷۲۰ دقیقه در ماه مصرف می‌شود (برای مخازن عمومی رایگان و برای خصوصی زیر سقف ۲۰۰۰ دقیقه).

## 🛠 مشکلات رایج

| مشکل | راه‌حل |
|------|--------|
| ربات جواب نمی‌دهد | توکن `TELEGRAM_TOKEN` درست است؟ مطمئن شوید در `Actions`، دسترسی Read & Write فعال باشد. یک بار `Run workflow` را دستی بزنید. |
| خطای ۴۰۳ (Forbidden) در ذخیره‌سازی | دسترسی Read & Write در `Settings > Actions > General` را بررسی کنید. |
| فایل‌ها روی Raw URL خطای ۴۰۴ می‌دهند | چند ثانیه صبر کنید (کمی تأخیر طبیعی است) و صفحه را رفرش کنید. |
| دانلود فایل خیلی بزرگ خطا می‌دهد | گیتهاب فایل‌های بالای ۱۰۰ مگابایت را مستقیم قبول نمی‌کند. |

## 📁 ساختار پروژه

```
.
├── .github/workflows/
│   ├── telegram_bot.yml       # ربات (پول هر ۵ دقیقه)
│   └── download.yml           # دانلود از طریق فایل محرک
├── bot.go                     # کد ربات
├── download.go                # کد دانلودر (فایل محرک)
├── trigger_download.txt       # فایل محرک دستی
└── downloads/                 # پوشه‌ی ذخیره‌ی فایل‌ها
```

## 💡 نکته‌ی پایانی

پروژه دقیقاً با API استاندارد ربات‌های تلگرام کار می‌کند، اما **اندپوینت** آن به `https://tapi.bale.ai/bot` تغییر داده شده تا روی بله اجرا شود.  
اگر خواستید از تلگرام استفاده کنید، فقط همین یک خط را در `bot.go` عوض کنید.
