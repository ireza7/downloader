# 📥 سیستم دانلود فایل با ربات تلگرام و گیتهاب اکشنز

<div dir="rtl">

**پروژه‌ای کامل برای دانلود فایل از لینک‌های عمومی و ذخیره در ریپازیتوری گیتهاب**  
با دو روش ساده:
1. **ویرایش فایل `trigger_download.txt`** – با سه بخش مختلف برای کنترل نحوه ذخیره‌سازی
2. **ربات تلگرام** – فقط با ارسال لینک، فایل‌ها را دانلود و لینک مستقیم دریافت کنید

ربات با تکنیک **Long Polling** تقریباً همیشه آنلاین است و امکانات زیر را دارد:

- دانلود **همه نوع فایل** با دنبال کردن خودکار تغییر مسیرها (Redirect)
- دانلود **موازی** (تا ۵ فایل هم‌زمان) برای افزایش سرعت
- سه حالت ذخیره‌سازی:
  - **ساده** – فایل‌ها بدون تغییر ذخیره می‌شوند
  - **zip کلی** – همه فایل‌ها در یک فایل فشرده
  - **zip جداگانه** – هر فایل در یک فایل فشرده جداگانه
- پشتیبانی کامل از **روش فایلی** با سه بخش مجزا
- مشاهده‌ی لیست فایل‌های دانلود شده از طریق ربات (`/list`)
- **بدون نیاز به هیچ سرور خارجی** – همه چیز روی گیتهاب اجرا می‌شود
- استفاده از **Long Polling** برای پاسخ‌دهی سریع (حداکثر تأخیر چند دقیقه)

---

## 🚀 راه‌اندازی در ۳ قدم

### ۱. فورک کردن پروژه
این ریپازیتوری را **Fork** کنید تا یک نسخه در اکانت گیتهاب خود داشته باشید.

### ۲. تنظیم متغیر مخفی (Secret)
به مسیر **Settings → Secrets and variables → Actions** بروید و **فقط یک راز** اضافه کنید:

| نام              | توضیح |
|------------------|-------|
| `TELEGRAM_TOKEN` | توکن ربات تلگرام (از [BotFather](https://t.me/BotFather) دریافت کنید) |

> 🟢 دیگر نیازی به ساختن Personal Access Token (PAT) نیست!  
> ورک‌فلوها از `GITHUB_TOKEN` پیش‌فرض گیتهاب با دسترسی کامل به مخزن استفاده می‌کنند.

### ۳. فعال‌سازی ورک‌فلوها
پس از فورک، ورک‌فلوها ممکن است غیرفعال باشند. یک بار از طریق زبانه **Actions** گیتهاب، `Telegram Bot (Hourly Long Poll)` را با `Run workflow` اجرا کنید تا cron فعال شود. همچنین می‌توانید یک Push کوچک (مثلاً ویرایش README) انجام دهید.

## 📂 روش استفاده

### 🔹 روش اول: ویرایش فایل `trigger_download.txt`

در ریشه پروژه، فایل `trigger_download.txt` سه بخش دارد. لینک‌های خود را زیر کامنت مناسب وارد کنید و سپس Commit بزنید.  
پس از چند ثانیه، ورک‌فلو اجرا شده و فایل‌ها در پوشه‌ی `downloads/` قرار می‌گیرند. فایل محرک خودکار به حالت اولیه باز می‌گردد.

#### حالت‌های موجود:

```text
# اگر لینک های خود را در زیر این کامنت وارد کنید فایل ها بدون تغییر به صورت ساده ذخیره میشوند

# اگر لینک هارا زیر این کامنت وارد کنید همه فایل ها در یک فایل فشرده زیپ ذخیره میشوند

# اگر لینک هارا زیر این کامنت وارد کنید هر فایل به صورت یک فایل فشرده زیپ ذخیره میشوند
```

**مثال:**
- برای دانلود دو عکس به صورت ساده:
  ```
  # اگر لینک های خود را در زیر این کامنت وارد کنید فایل ها بدون تغییر به صورت ساده ذخیره میشوند
  https://example.com/photo1.jpg
  https://example.com/photo2.jpg
  ```
- برای دانلود همان دو عکس در یک فایل zip:
  ```
  # اگر لینک هارا زیر این کامنت وارد کنید همه فایل ها در یک فایل فشرده زیپ ذخیره میشوند
  https://example.com/photo1.jpg
  https://example.com/photo2.jpg
  ```
- برای فشرده‌سازی جداگانه هر فایل:
  ```
  # اگر لینک هارا زیر این کامنت وارد کنید هر فایل به صورت یک فایل فشرده زیپ ذخیره میشوند
  https://example.com/photo1.jpg
  https://example.com/photo2.jpg
  ```

⚠️ توجه: لینک‌های قرار داده شده زیر هر بخش، فقط مطابق همان حالت پردازش می‌شوند. می‌توانید هم‌زمان از چند بخش استفاده کنید.

### 🔹 روش دوم: ربات تلگرام

پس از راه‌اندازی ربات، در تلگرام به آن پیام دهید. امکانات:

| دستور / عملکرد | توضیح |
|---------------|-------|
| `/start`      | پیام خوش‌آمدگویی |
| `/help`       | راهنمای کامل |
| `/simple` + لینک(ها) | دانلود همه فایل‌ها به صورت ساده |
| `/zipall` + لینک(ها) | همه فایل‌ها در یک zip |
| `/zipeach` + لینک(ها)| هر فایل در یک zip جداگانه |
| `/list`       | نمایش لیست فایل‌های موجود در ریپو |
| ارسال مستقیم لینک (بدون دستور) | **تک لینک:** دانلود ساده – **چند لینک:** همه در یک zip |

**مثال‌ها:**
- دانلود ساده یک فایل:  
  `https://example.com/file.pdf`
- دانلود چند فایل در یک zip:  
  `https://example.com/1.jpg`  
  `https://example.com/2.jpg`
- فشرده‌سازی هر عکس جداگانه:  
  `/zipeach https://example.com/1.jpg https://example.com/2.jpg`

📥 ربات بلافاصله پس از دانلود، لینک مستقیم فایل(ها) را از `raw.githubusercontent.com` ارسال می‌کند.

## ⚙️ جزئیات فنی

- **زبان برنامه‌نویسی**: Go (گولنگ)  
- **مدیریت اجرا**:
  - ربات با `concurrency` تنظیم شده تا همیشه یک نمونه فعال باشد.
  - هر **۱ ساعت** یک بار یک اکشن شروع می‌شود و با **Long Polling** تا **۵۰ دقیقه** آنلاین می‌ماند (وقفه‌ای حدود ۱۰ دقیقه بین اجراها وجود دارد).
  - در هنگام آنلاین بودن، پاسخ‌دهی تقریباً آنی است (تحت ۳۰ ثانیه).
- **دانلود موازی**: حداکثر ۵ دانلود هم‌زمان.
- **پشتیبانی از Redirect**: تمام لینک‌ها تا ۱۰ مرحله دنبال می‌شوند.
- **محدودیت حجم فایل**: گیتهاب برای فایل‌های بالای ۱۰۰ مگابایت محدودیت دارد. این پروژه برای فایل‌های بزرگ مناسب نیست.

```
├── trigger_download.txt       # فایل محرک (ویرایش و کامیت)

```

## 🛠 عیب‌یابی

- **ربات پاسخ نمی‌دهد**: مطمئن شوید `TELEGRAM_TOKEN` صحیح است و یک بار `workflow_dispatch` روی `Telegram Bot` اجرا کرده‌اید.
- **فایل در ریپو ظاهر نمی‌شود**: ممکن است تا چند ثانیه تأخیر داشته باشد. صفحه را refresh کنید.
- **خطای دسترسی (۴۰۳)**: در صورت عدم استفاده از فایل محرک، دسترسی ربات از طریق `GITHUB_TOKEN` (نه PAT) انجام می‌شود. اطمینان حاصل کنید که در تنظیمات Actions، `Workflow permissions` روی **Read and write permissions** تنظیم شده است.
- **برای ریپازیتوری خصوصی**: مصرف گیتهاب اکشن‌ها در نظر داشته باشید. اجرای ساعتی با ۵۰ دقیقه فعال بودن، حدود ۳۶۰۰۰ دقیقه در ماه مصرف می‌کند که از سقف رایگان ۲۰۰۰ دقیقه فراتر است. برای خصوصی می‌توانید cron را به بازه‌های طولانی‌تر (مثلاً هر ۶ ساعت) تغییر دهید.
